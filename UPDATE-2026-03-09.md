本次更新主要完成了 BSC 路由升级、Flap 协议接入、CREATE2 靓号部署，以及插件侧的统一报价与授权逻辑改造。这也是一次偏安全与稳定性的版本升级。

  - Proxy / Impl 均已完成 BscScan 源码验证
  - 新增 CREATE2 靓号部署流程：`prepare-vanity.js` + `deploy-vanity.js`

## 安全性重点

- **私钥不上传服务器**：这是本次版本最应该强调的安全原则。私钥只在本地浏览器插件后台解密和使用，不依赖中心化后端托管
- **前端页面拿不到明文**：普通 UI 页面不接触私钥明文，明文只在本地后台内存中短暂存在，用于签名和发送交易
- **私钥本地加密**：插件继续使用 `PBKDF2 + AES-256-GCM` 进行本地加密，密钥不落盘，默认通过密码解锁后仅在本地内存中短暂存在
- **自动锁定**：支持自动锁定，超时后清空内存中的派生密钥，降低长期解锁暴露面
- **交易时效保护**：BSC 路由 `buy` / `sell` 统一增加 `deadline`，避免交易长时间悬挂后被延迟执行
- **授权目标链上返回**：插件不再本地猜测 approve 目标，改用合约返回的 `approveTarget`，减少错误授权风险
- **公开 RPC 风险提示**：文档中明确建议使用隐私防夹 RPC，降低公开 mempool 被夹子的概率
- **源码公开可审计**：FreedomRouter Proxy 与 Impl 已完成 BscScan 验证，当前链上逻辑可直接审阅
